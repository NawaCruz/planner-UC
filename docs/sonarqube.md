# SonarQube

## 6. Actividades Tecnicas Detalladas

## 6.1. Evaluacion de Calidad de Codigo mediante SonarQube

### a. Configuracion

1. Instalar y configurar SonarQube.
2. Integrar SonarQube con el repositorio GitHub del proyecto.
3. Configurar analisis automatico del proyecto.

### b. Analisis Estatico

Se debera ejecutar un analisis completo del codigo fuente del Frontend y Backend considerando:

1. Bugs.
2. Vulnerabilities.
3. Code Smells.
4. Duplicacion de codigo.
5. Maintainability Rating.
6. Reliability Rating.
7. Security Rating.
8. Technical Debt.
9. Cobertura de pruebas.

### c. Interpretacion Tecnica

1. Analizar metricas obtenidas.
2. Identificar componentes criticos.
3. Justificar problemas detectados.
4. Proponer mejoras sustentadas tecnicamente.
5. Implementar correcciones verificables.

### d. Evidencias Obligatorias

1. Capturas de dashboard.
2. Metricas antes y despues de correcciones.
3. Reporte tecnico de analisis.
4. Evidencia de reduccion de deuda tecnica.
